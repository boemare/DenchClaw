Platform

Built on OpenClaw

OpenClaw provides the skills engine, model gateway, and runtime. Eve adds event-specific logic — venue sourcing, outreach, booking, and publishing — through a dedicated profile and web application.

Web ApplicationNext.js 15, React 19
Chat-based web UI streaming responses via SSE. Displays call transcripts inline, shows email threads, and surfaces approval gates.
OpenClaw RuntimeAgent Platform
Model routing via AI Gateway, skills registry for pluggable modules, session persistence, memory, and workspace management. Eve runs on an isolated profile with its own state directory.
Skills Layer8 Pluggable Modules
Each integration is a self-contained skill with its own behavior rules. The orchestration skill coordinates subagents — browser research, Retell calls, AgentMail threads, and Eventbrite events — in parallel or sequence.
Local StorageDuckDB + JSONL
All data stays local. DuckDB for structured CRM data. JSONL for call transcripts and email threads. JSON config, encrypted credentials.

Privacy

Data Boundary Design

Eve talks to venues, never to fans. All fan-facing communication goes through Eventbrite. No PII flows through Eve's pipeline.

What Eve Touches

  • Venue names, addresses, phone numbers (business data)
  • Pricing quotes and availability (operational data)
  • Aggregate RSVP counts from Eventbrite (no attendee details)
  • Team member messages and approvals (internal comms)

What Eve Never Touches

  • Fan email addresses or phone numbers
  • Attendee lists or personal details
  • Payment card data (invoices go to finance team)
  • Direct messages to fans — Eventbrite handles this
Privacy LayerPII Protection Skill
Sits between Eve and all outputs, filtering and redacting sensitive data from logs, transcripts, and API calls. All data stored locally. Human approval required before any purchase or booking.

Integrations

External Services

Each integration is a self-contained OpenClaw skill with its own credentials and behavior rules.

Retell AI — Voice Calls

Two agents (discovery + booking) make outbound calls. Eve provides venue details, date, headcount, and budget. Returns transcripts, call analysis, and recordings — logged to DuckDB and rendered in the chat UI.

AgentMail — Email

Dedicated inbox (eve@agentmail.to) with automatic conversation threading. Persisted as JSONL sessions, displayed alongside call transcripts.

Eventbrite — Events

Creates fan-facing event page with tickets, venue, and timing. Eventbrite handles attendee comms (confirmations, reminders, waitlist). Eve reads RSVP counts only — never attendee PII.

DuckDB — Local CRM

Local embedded database storing events, venues, comms log, and attendee counts. Queryable via workspace API. No cloud — everything stays on-machine.
